# 글로벌 오더 개발 연동 가이드

이 가이드에서는 글로벌 오더 시스템과 연동하기 위한 SDK 사용 방법 및 상품 데이터 구성을 안내합니다. 단계별 설명을 통해 SDK를 효율적으로 적용할 수 있도록 돕겠습니다.

---

## 📥 SDK 사용 방법

### 1. SDK 로드하기
먼저, HTML 페이지에 SDK를 포함하여 연동을 시작합니다. 아래와 같은 스크립트를 `<head>` 또는 `<body>` 태그 안에 추가하세요:

```html
<script src="https://sdk-checkout-staging.delivered.co.kr/index.js"></script>
```

### 2. 주문 생성하기
SDK가 로드되면 `dkLibrary.order` 메서드를 이용하여 주문을 생성할 수 있습니다. `CartProductRequest` 배열을 설정하여 장바구니에 상품 정보를 전달하세요:

```typescript
const cartProducts: CartProductRequest[] = [
  // 여기에 상품 데이터를 추가하세요
];

// 예시: 주문 생성
dkLibrary.order(cartProducts);
```

## 📋 데이터 타입 정의

주문 데이터를 구성하기 위한 각 타입을 정의합니다. 아래에 설명된 타입을 참고하여 올바른 형식으로 데이터를 준비하세요.

### 1. 상품 옵션 데이터: `OptionRequest`
각 상품의 옵션은 `OptionRequest` 인터페이스를 통해 표현됩니다.

```typescript
interface OptionRequest {
  title: string; // 옵션의 이름
  price: number; // 옵션 가격
}
```

### 2. 상품 데이터: `ProductRequest`
상품에 대한 정보를 `ProductRequest` 타입으로 구성합니다. 아래 속성들을 포함하며, 상세 설명을 참고하세요:

```typescript
interface ProductRequest {
  sellerId?: number;    // 선택 사항: DK에서 제공한 seller ID
  title: string;        // 상품 이름
  price: number;        // 상품 기본 가격
  originUrl: string;    // 상품이 게시된 URL
  images: string[];     // 상품 이미지 URL 배열
  options: OptionRequest[]; // 모든 옵션 데이터
}
```

#### **📌 필수 요건**
- **`sellerId`**: 선택 항목입니다. 해당 값이 있을 경우, 헤더에 seller 로고와 약관 동의가 표시됩니다.
- **`options`**: 옵션이 없는 상품이라도 `options` 배열은 1개 이상의 항목을 포함해야 합니다. 옵션이 없는 경우, 기본 값으로 `{ title: "", price: 0 }`을 설정하세요.

### 3. 선택된 옵션 데이터: `SelectedOptionRequest`
구매자가 선택한 옵션 정보를 `SelectedOptionRequest` 인터페이스로 정의합니다:

```typescript
interface SelectedOptionRequest {
  title: string;   // 선택한 옵션 이름
  quantity: number; // 선택한 옵션 수량
}
```

#### **📌 필수 요건**
- **`title`**: `ProductRequest`의 `options`에 동일한 title이 존재해야 합니다. 

### 4. 장바구니 상품 데이터: `CartProductRequest`
최종적으로, 장바구니에 추가할 상품과 옵션 데이터를 `CartProductRequest` 인터페이스로 구성합니다:

```typescript
interface CartProductRequest {
  productRequest: ProductRequest;          // 상품 데이터
  selectedOptions: SelectedOptionRequest[]; // 선택된 옵션 데이터
}
```

---

## 📌 참고 사항

- **최종 가격 계산**: 최종 상품 가격은 `상품 가격 + 옵션 가격`으로 계산됩니다.
- **옵션이 없는 경우**: `ProductRequest`의 `options` 배열은 최소 1개의 항목을 가져야 하므로, 옵션이 없는 상품이라면 `{ title: "", price: 0 }`을 추가합니다.
- **선택 옵션 전달 형식**: 구매자가 선택한 옵션과 수량은 아래와 같은 형식으로 전달합니다:
  
  ```typescript
  selectedOptions: [{ title: "", quantity: 3 }]
  ```

올바른 데이터 형식으로 상품 정보를 구성한 후, `dkLibrary.order(cartProducts);`를 호출하여 주문을 완료하세요. 위의 설명을 참고하여 원활한 연동을 진행하시기 바랍니다.
